# Морской бой 98 — онлайн

PvP-версия «Морского боя» в стиле Windows 98. Быстрый матч со случайным
соперником или своя комната по коду для игры с другом. Ручная расстановка
флота кликом (с поворотом) или кнопка «Авто». Классический флот: 1×4, 2×3,
3×2, 4×1 (20 клеток), попал — стреляешь ещё раз.

Это отдельный сервис от «Шпион 98» — свой код, своя игровая логика,
свой деплой. Общее между ними — только стиль и, в будущем, единый
Telegram-вход (десктоп с иконками обеих игр).

## Запуск локально

```bash
npm install
node server.js        # http://localhost:3000
```

Открой в двух вкладках — в одной "Создать комнату для друга", в другой
"Войти по коду", либо в обеих сразу нажми "Быстрая игра".

## Настройки (переменные окружения)

| Переменная | По умолчанию | Что это |
|---|---|---|
| PORT | 3000 | порт HTTP/WS |
| T_PLACE | 90 | секунд на расстановку флота |
| T_SHOT | 45 | секунд на выстрел (иначе ход переходит сопернику) |

## Тест

```bash
PORT=3200 T_PLACE=20 T_SHOT=10 node server.js &
node test/e2e.js   # PASS = быстрый матч сыгран от начала до победы
```

## Деплой на Railway (отдельный сервис)

1. Создай новый репозиторий на GitHub (например `battleship-online`) — отдельно
   от `spy98-online`, не смешивай.
2. Залей туда всё содержимое этой папки: `server.js`, `lib.js`, `package.json`,
   `package-lock.json`, `public/`, `test/` (без `node_modules`).
3. В Railway: New Project → Deploy from GitHub repo → выбери `battleship-online`.
4. Settings → Networking → Generate Domain.
5. Если Railway не найдёт entrypoint сам — Set start command → `node server.js`.

## Подключение к Telegram

Аналогично Шпиону: BotFather → `/newapp`, привязать к HTTPS-домену.
Ссылки-приглашения: `https://t.me/<bot>/<app>?startapp=КОД`.

## Протокол (клиент → сервер)

`create{nick}` `join{code,nick}` `quick{nick}` `rejoin{code,token}`
`place{cells}` / `place{auto:true}` `shoot{idx}`

Сервер → клиент: `joined` `queued` `state` `log` `result` `err`.

Комната — машина состояний: `lobby(2 игрока) → placement → battle → over`.
Игровая логика (валидация расстановки, авто-расстановка) вынесена в `lib.js` —
без сети, легко тестировать отдельно.
